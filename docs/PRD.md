# 產品需求文件 (PRD) - 個人記帳簿系統

## 1. 專案概述
- **背景與動機**：大學生通常有固定零用錢或打工作為收入來源，但常常不清楚錢花在哪裡。市面上的記帳軟體功能往往過於複雜，導致難以養成記帳習慣。因此，需要一款極簡、專注於日常收支追蹤的系統。
- **目標用戶**：大學生
- **核心價值主張**：極簡操作介面，快速紀錄收支，幫助大學生輕鬆掌握財務狀況，無需複雜學習即可建立記帳習慣。

## 2. 功能需求
作為核心功能，系統將包含以下項目：

1. **快速記錄收支**
   - 作為大學生，我希望能夠在幾秒鐘內輸入金額、分類和備註並儲存，以便快速完成日常記帳，不耽誤時間。
2. **餘額自動計算**
   - 作為大學生，我希望系統能自動加總收入與支出並顯示當前餘額，以便我隨時知道還剩下多少錢可以花。
3. **收支分類統計**
   - 作為大學生，我希望可以將收支分類（如飲食、交通、娛樂等），以便月底時了解各項花費的比例。
4. **歷史紀錄查詢與修改**
   - 作為大學生，我希望能夠查看過去的收支明細，並能修改或刪除，以便更正錯誤輸入的紀錄。
5. **月度報表總覽**
   - 作為大學生，我希望能在首頁看到當月的總收入、總支出報表，以便對整月的財務狀況一目了然。

## 3. 非功能需求
- **技術限制**：使用 Flask + Jinja2 + SQLite 進行開發。
- **效能與安全考量**：
  - 系統響應時間應盡可能短，確保「快速記帳」的流暢體驗。
  - 具備基本的資安防禦機制（如防止 SQL Injection、XSS），保護資料安全。

## 4. MVP 範圍
- **Must Have (必須有)**：快速記錄收支、餘額自動計算、歷史紀錄明細查詢、基本的收支分類。
- **Should Have (應該有)**：可修改/刪除已存在的紀錄、基本的統計圖表。
- **Nice to Have (可以有)**：預算超支提醒、匯出報表功能、自訂個人專屬分類。
